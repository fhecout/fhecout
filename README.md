## 👨‍💻 Sobre mim

Sou **desenvolvedor back-end** focado em construir **APIs, automações e soluções eficientes** para manipulação e organização de dados. Trabalho principalmente com **Node.js, Python, JavaScript e PostgreSQL**, buscando escrever código bem estruturado, escalável e fácil de manter.

Nos últimos meses venho me aprofundando em **IA aplicada** — desde os fundamentos de machine learning e NLP até engenharia de prompt e protocolos de integração de agentes (MCP) — e aplicando esse conhecimento em projetos próprios de automação e agentes locais.

- 🔭 Atualmente construindo o **Nautilus**, um agente de IA pessoal local-first.
- 🌱 Aprofundando conhecimentos em **IA aplicada, prompt engineering e arquitetura de agentes**.
- 📍 Baseado em Curitiba, Brasil.
- 💬 Aberto a oportunidades e colaborações em back-end e projetos com IA.

## 💡 Projeto em Destaque

<details open>
<summary><b>🧭 Nautilus — Agente de IA pessoal, local-first</b></summary>
<br/>

Agente de IA que roda localmente via **Ollama**, com interface desktop em **Electron + React**, execução de ferramentas no computador, memória persistente e um sistema auditável de decisões.

| | |
|---|---|
| **Stack** | Node.js 22 (ESM), Ollama, Express 5, React 19 + Vite, Electron 42, SQLite nativo |
| **Destaques** | Tool calling modular, streaming via SSE, Agent Team com subagentes especializados (Architect, Security, Research, Executor, Critic, Decision) |
| **Segurança** | "Safe Mode" para bloquear operações destrutivas até confirmação explícita |
| **Observabilidade** | "Black Box" em SQLite registrando runs, eventos e tool calls |
| **Automação** | Rotinas agendadas (briefing diário, revisão semanal, auditoria de Safe Mode) |
| **Repositório** | [github.com/fhecout/Nautilus](https://github.com/fhecout/Nautilus) |

O projeto inclui um "Problem Radar" que varre o código em busca de arquivos grandes demais, ferramentas sem teste e dependências ociosas, além de um modo "Decision Room" onde subagentes especializados deliberam sobre decisões técnicas antes de consolidar uma resposta final.

</details>
